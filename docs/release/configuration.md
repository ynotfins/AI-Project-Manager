## Security Notes (Read First)

**Secrets are never stored in any committed file.** All credentials are injected at runtime:

1. **Bitwarden Secrets Manager** (`bws` CLI) — source of truth for LLM API keys
2. **Bitwarden Password Manager** (`bw` CLI) — stores machine account access tokens
3. **WSLENV** — passes secrets from Windows PowerShell process into WSL at startup
4. **Transient `.gateway-env`** — written to `~/.openclaw/.gateway-env` (chmod 600) only during systemd gateway restart; deleted within 8 seconds
5. **Windows User Env Vars** (`HKCU\Environment`) — used for DroidRun keys; encrypted by Windows, survives reboots
6. **systemd EnvironmentFile** — `~/.config/systemd/user/openclaw-gateway.service.d/api-keys.conf` loads from `.gateway-env` (transient)

### Secret Delivery Flow

```
Bitwarden Secrets Manager
  └── ANTHROPIC_API_KEY, OPENAI_API_KEY, OPENROUTER_API_KEY (OpenClaw project)
  └── DEEPSEEK_API_KEY, OPENROUTER_API_KEY (DroidRun project)
       │
       ▼ (bws run ... / bw get item)
PowerShell environment ($env:VAR) — in-memory only
       │
       ├─► WSLENV → transient ~/.openclaw/.gateway-env
       │       → systemd EnvironmentFile → openclaw-gateway process
       │       → file deleted after 8 seconds
       │
       └─► Windows HKCU\Environment (DROIDRUN_DEEPSEEK_KEY, DROIDRUN_OPENROUTER_KEY)
               → read by mcp_server.py at tool invocation time
```

## open--claw — Environment Variables

**Config precedence (highest → lowest):**
1. Process environment (parent shell / systemd EnvironmentFile)
2. `.env` in working directory (non-overriding)
3. Global `~/.openclaw/.env` (non-overriding)
4. Config `env` block in `~/.openclaw/openclaw.json`
5. Optional login-shell import (if `env.shellEnv.enabled`)

### LLM Provider Keys (Required for AI responses)

| Variable | Purpose | Default | Rotation | Where Set |
|----------|---------|---------|----------|-----------|
| `ANTHROPIC_API_KEY` | claude-sonnet-4-5 / claude-opus-4-6 (primary tier) | None — must be set | Rotate in Anthropic Console → update in Bitwarden Secrets → re-run startup script | Bitwarden Secrets Manager (OpenClaw project) → WSLENV → `.gateway-env` → systemd |
| `OPENAI_API_KEY` | gpt-4o / gpt-4o-mini (fallback + budget tier) | None — must be set | Rotate in OpenAI Console → update in Bitwarden Secrets → re-run startup script | Same as above |
| `OPENROUTER_API_KEY` | Multi-provider routing (optional fallback) | None | Rotate in OpenRouter Console → Bitwarden Secrets | Same as above |

### Channel Credentials (Low-risk, in ~/.openclaw/.env)

| Variable | Purpose | Default | Storage | Notes |
|----------|---------|---------|---------|-------|
| `TELEGRAM_BOT_TOKEN` | Telegram bot authentication | None | `~/.openclaw/.env` (WSL, not committed) | Get from @BotFather. Allowed exception to Bitwarden rule — low-risk channel token |
| `OPENCLAW_GATEWAY_TOKEN` | Internal gateway auth token | Auto-generated by `openclaw doctor` | `~/.openclaw/openclaw.json` → `gateway.auth.token` | Managed by OpenClaw; NOT a user-provided credential |

### Gateway Runtime Markers (Injected by OpenClaw — not user-configurable)

| Variable | Value | Purpose |
|----------|-------|---------|
| `OPENCLAW_SHELL` | `exec` / `acp` / `acp-client` / `tui-local` | Context marker in spawned child processes |
| `OPENCLAW_LOAD_SHELL_ENV` | `1` | Enable login-shell env import |
| `OPENCLAW_SHELL_ENV_TIMEOUT_MS` | `15000` | Timeout for shell env import |

## API Key Rotation Process

### Rotating an OpenClaw LLM Key

1. Generate new key in provider console (Anthropic / OpenAI / OpenRouter)
2. Open Bitwarden Secrets Manager → project "OpenClaw" → update secret value
3. Re-run startup script: `bws run --project-id f14a97bb-... -- pwsh -NoProfile -File "$HOME\.openclaw\start-cursor-with-secrets.ps1"`
4. Verify: startup output shows `ENV_CHECK: <KEY> = SET`
5. Verify gateway health: `wsl -- bash -lc "source ~/.nvm/nvm.sh && cd ~/openclaw-build && pnpm openclaw health"`

### Rotating a DroidRun Key

1. Generate new key in provider console (DeepSeek / OpenRouter)
2. Open Bitwarden Secrets Manager → project "DroidRun" → update `DEEPSEEK_API_KEY` or `OPENROUTER_API_KEY`
3. Re-run `.\scripts\store_api_keys_to_env.ps1` (or full startup script)
4. Verify: `$env:DROIDRUN_DEEPSEEK_KEY` contains the new key value

### Rotating Telegram Bot Token

1. Message @BotFather on Telegram → `/revoke` → copy new token
2. Update `~/.openclaw/.env` in WSL: `TELEGRAM_BOT_TOKEN=<new-token>`
3. Restart gateway: `systemctl --user restart openclaw-gateway.service`

### Rotating Android Signing Key

> Only needed if building release APK

1. Generate new keystore or rotate key alias
2. Update `~/.gradle/gradle.properties` with new values
3. Rebuild: `build_apk_release.bat`
